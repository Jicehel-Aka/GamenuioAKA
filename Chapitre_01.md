# 🎯 Objectif du chapitre
Ce premier chapitre introduit le projet : créer un **casse‑briques complet, optimisé et extensible** sur la **Gamebuino AKA**, en suivant un tutoriel structuré en 22 chapitres + un bonus.

---

# 🕹️ Présentation du projet
Vous allez développer un jeu comprenant :

- une boucle de jeu stable  
- une machine à états  
- une physique Arkanoid  
- des briques destructibles et incassables  
- des bonus (colle, laser, multi‑ball)  
- un système audio professionnel  
- une sauvegarde SD  
- des menus  
- une génération procédurale  
- des optimisations réelles  
- des particules, un boss, des shaders, des sprites animés  

---

# 🎮 Pourquoi la Gamebuino AKA ?
La Gamebuino AKA est une console :

- compacte  
- programmable en C++  
- basée sur ESP32‑S3  
- dotée d’un écran 320×240  
- avec un SDK moderne  
- idéale pour apprendre l’architecture de jeu  

---

# 📘 Ce que vous allez apprendre
- architecture logicielle  
- gestion des entrées  
- rendu graphique  
- audio temps réel  
- collisions  
- optimisation  
- structuration du code  
- publication GitHub

- Vous allez apprendre des techniques utilisées dans les vrais moteurs de jeu.
